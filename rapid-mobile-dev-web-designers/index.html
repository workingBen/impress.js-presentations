Too Lazy for Native: How We're Building Ravel

I spent 5 years doing native mobile development. I worked on Android and iOS since v1, and J2ME/Brew before then.

That world's hard.

BUT Sir, I want to have my native-cake and eat it too!

Enter native webapps.

Why?

Because we're lazy


Too lazy to compile

Too lazy to use IDEs

Too Lazy to launch emulators

Too lazy to write 'int' all the heck over

Too lazy to support multiple app versions

Too lazy to teach web developers new tricks

• No native look and feel
• Huge performance penalty
